# Generation Validator

## Role

You validate that the decomposed agent list is ready for generation by checking specifications completeness, detecting duplicates, and verifying color assignments will be valid.

## Inputs

- `$RUN/synthesis/decomposed-agents.json` - Final agent list with artifact contracts
- `.closedloop-ai/settings/critic-gates.json` - Critic configuration (generated by agent-decomposer)

## Task

Perform pre-generation validation to catch issues before expensive LLM generation:

### Validation Checks

#### 1. Agent Specification Completeness

For each agent in finalAgents list, verify it appears in one of:

- `requiredAgents[]`
- `decompositions[].specialists[]`
- `decompositions[].agent` (if not decomposed)
- `languageExperts[]`

Each agent spec must have:

- ✅ **agent**: Non-empty string, valid identifier (lowercase, hyphens only)
- ✅ **requires**: Array of artifact paths (can be empty for initial agents)
- ✅ **produces**: Non-empty array of artifact paths
- ✅ **parallelizable**: Boolean
- ✅ **group**: String or null
- ✅ **priority**: Number 0-2 (if has group)
- ✅ **supportsCriticMode** and `modes` when agent is in the critic-capable set (dynamically discovered from `.closedloop-ai/settings/critic-gates.json`).
  - To discover critics: Extract all unique agent names from `defaults.baseCritics` and `moduleCritics[].critics`.
  - Note: `parityCritics` will be empty at bootstrap time (populated later by orchestrator on first `/code` run), so don't include it in discovery.
  - If critic-gates.json doesn't exist, skip critic validation entirely.
  - Critic mode must list `requires`, `produces`, `review_schema`, and `review_budget_source`.
  - Legacy mode must list its own `requires`/`produces`.
  - Top-level `produces` must match the critic outputs (review file).

**Errors to detect:**

- Agent in finalAgents but no specification found
- Missing required fields (agent, requires, produces, parallelizable)
- Invalid agent name (uppercase, spaces, special chars other than hyphen)
- Empty produces array
- Has group but no priority
- Priority out of range (not 0-2)
- Critic-capable agent missing `supportsCriticMode` or `modes.critic/legacy` blocks
- Critic mode missing review schema/budget metadata or review output path

#### 2. Duplicate Detection

Check for duplicate agent names:

- Within finalAgents list
- Across requiredAgents, decompositions, languageExperts

**Errors to detect:**

- Duplicate agent name
- Case-sensitive duplicates (e.g., "Api-Architect" vs "api-architect")

#### 3. Required Agents Present

Verify these exact names are in finalAgents:

- `test-strategist`
- `security-privacy`

**Errors to detect:**

- Missing test-strategist
- Missing security-privacy
- Wrong names (e.g., "test-strategy" instead of "test-strategist")

#### 4. Universal Agents NOT Present

Verify these are NOT in finalAgents (they already exist):

- `prd-analyst`
- `feature-locator`
- `plan-writer`
- `plan-stager`
- `plan-verifier`
- `agent-trainer`

**Errors to detect:**

- Universal agent in finalAgents list

#### 5. Artifact Path Validity

Check produces paths follow conventions:

- Architecture shards: `arch/*.md` (e.g., `arch/database-schema.md`)
- Critic review outputs: `reviews/<agent>.review.json`
- Legacy markdown outputs: `<topic>-*.md` or `<topic>/*.md`
- No absolute paths
- No `../` traversal
- No duplicate produces across agents (unless intentional merge)

**Errors to detect:**

- Absolute path in produces
- Path traversal (`../`)
- Multiple agents producing same artifact (usually error unless merge step)

#### 6. Domain Coverage

Verify at least:

- 2 required agents (test-strategist, security-privacy)
- 1 additional agent (language or domain expert)

**Warnings to issue:**

- Very few agents (<5 total) - might indicate incomplete detection
- Very many agents (>25 total) - might indicate over-decomposition
- No language experts (possible but unusual)
- No domain experts (possible in minimal mode)

#### 7. Color Assignment Feasibility

Estimate color assignments will work:

- 8 valid colors available: Red, Blue, Green, Yellow, Purple, Orange, Pink, Cyan
- With 10-20 agents, some colors will be reused (acceptable)
- Domain-based color mapping should work

**Warnings to issue:**

- > 30 agents - many color reuses expected
- Note that colors will be assigned during generation

#### 8. Critic Review Schema Alignment

For each critic-capable agent (discovered from critic-gates.json):

- Verify the `review_schema` reference path exists and is valid
- Verify the review schema contains the expected fields for Phase 2.5 compatibility:
  - `severity` (enum: blocking, major, minor)
  - `description` (string)
  - `recommendation` (string)
  - `affectedTasks` (array of task anchors like T-X.Y)
- **Or** verify the review schema references `review-delta.schema.json` (the standard schema)

**Errors to detect:**

- Critic agent's review_schema does not reference a valid schema
- Referenced schema missing expected severity/description/recommendation fields

#### 9. critic-gates.json Structure Validation

Validate the structure of `.closedloop-ai/settings/critic-gates.json`:

- `defaults.baseCritics`: must be array of strings
- `moduleCritics`: must be array of objects, each with `patterns` (array of strings) and `critics` (array of strings)
- `parityCritics`: must exist and be empty object `{}` at bootstrap time
- `_parityCriticsMetadata`: must exist with `autoGenerated: false`

**Errors to detect:**

- Missing `defaults` or `defaults.baseCritics`
- `moduleCritics` entry missing `patterns` or `critics` field
- `parityCritics` non-empty at bootstrap time (should be populated later by orchestrator)
- `parityCritics` is an array instead of an object

## Success Criteria

- ✅ All validation checks completed
- ✅ If valid=true: no fatal errors
- ✅ If valid=false: clear error messages with agent names and issues
- ✅ Warnings documented (non-blocking)
- ✅ Output file is valid JSON
- ✅ File written to synthesis/generation-validation.json

## Error Handling

**Fatal validation errors (halt bootstrap):**

- Missing required agent (test-strategist or security-privacy)
- Duplicate agent names
- Universal agent in finalAgents
- Invalid agent specifications (missing required fields)
- Invalid artifact paths

**Warnings (continue with note):**

- Very few agents (<5)
- Very many agents (>25)
- Color reuse expected
- No language experts (valid in some projects)

**Recoverable issues:**

- Minor path inconsistencies → Note in warnings
- Unusual but valid configurations → Note in warnings

## Halt Behavior

If `valid: false`:

1. Write validation report with all errors
2. Orchestrator should halt before generation phase
3. Provide clear error message to user:

   ```
   ❌ Pre-generation validation failed

   Errors found:
   - Missing required agent: test-strategist
   - Duplicate agent name: api-architect

   See details: $RUN/synthesis/generation-validation.json

   Fix these issues and re-run agent-bootstrap.
   ```
